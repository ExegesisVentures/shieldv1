═══════════════════════════════════════════════════════════════
  HOW IT WORKS
═══════════════════════════════════════════════════════════════

DUAL-LAYER ARCHITECTURE:

Layer 1: Hardcoded Registry (Instant)
────────────────────────────────────────────────────────────────
File: utils/coreum/token-registry.ts
➤ Instant client-side LP token detection
➤ No database query needed
➤ Already has ROLL/CORE LP token
➤ Used by: getLpPairInfo(), isLpToken()

Layer 2: Database (Scalable)
────────────────────────────────────────────────────────────────
Table: coreum_tokens with LP token columns
➤ Scalable, dynamic LP token management
➤ Queried via API endpoints
➤ Used for token search and listings

DETECTION FLOW:
────────────────────────────────────────────────────────────────
User views portfolio with LP tokens
    ↓
TokenTable.tsx: const lpPairInfo = getLpPairInfo(denom)
    ↓
token-images.ts → token-registry.ts (hardcoded lookup)
    ↓
Returns: {
  token0Symbol: "ROLL",
  token1Symbol: "CORE",
  token0Logo: "/tokens/roll.png",
  token1Logo: "/tokens/core.svg"
}
    ↓
UI renders: <DualTokenLogo {...lpPairInfo} />
    ↓
User sees: Overlapping ROLL + CORE logos
